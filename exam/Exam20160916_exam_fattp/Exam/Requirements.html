Exam management

Develop a program for managing university exams.
The classes belong to the package exams; the main class is ExamHandling. Class Example in package main provides examples of usage of the main methods and of the required checks.
In all cases the exceptions are of type ExamException.

The JDK documentation is availbale on a local server.

R1: Study plans and students

The method addStudyPlan(String name, String... courses) adds a study plan; the first argument is the name of the plan and the following arguments are the names of the courses that compose it.
A course is considered as defined when it appears for the first time in a study plan; the same course can be part of several study plans.
The method throws an exception if the plan name is duplicate.

The method enrollStudent(String studentId, String studentName, String studyPlan) records a new student with the given identifier and name, and links her to a study plan.
The method throws an exception in case of duplicate student id or undefined study plan.

The method numberOfStudentsForStudyPlan() returns the number of students for each study plan. The plans are sorted by name.

The method studentsForStudyPlan() returns the sorted list of students for each study plan; the plans are sorted by name.

R2: Exams

The method openExam(String courseName) opens an exam session for the given course. It throws an exception if the course is undefined or if there is already and open session for the course.

A student sustaining an exam is recorded through method takeExam(String studentId, String course).
The method throws an exception if the course is undefined, is not in the study plan of the student, there is no exam session open, the student already sustained the exam in the current session, or the student already holds a pass grade (i.e. >= 18) for the course.

The method studentsWhoTookTheExam(String courseName) returns the sorted list of the ids of the students that sustained the exam in the current session, i.e. for whom a takeExam() was called. It throws an exception if the course is undefined.

The method numberOfOpenExamSessions() returns the number of exam sessions currently (at the time of the invocation) open.

R3: Grading

The method giveGrade(String courseName, String studentId, int grade) allows assigning a grade to a student. The method throws an exception if the student did not sustain the exam in the currently open session (i.e. there was no takeExam() for the student), she already has a pass grade (>=18) for the course, or if the grade is not included in the interval 12 to 30 (extremes included).
In general a grade greater or equal to 18 is considered a pass grade, otherwise it is considered a fail grade.

The method closeExam(String courseName) closes an exam session for a course. It throws an exception if there is no open session for the course or if any student who sustained the exam did not received a grade.

After the closure of a session the method studentsWhoTookTheExam returns an empty list.

R4: Stats

The method gradesOfStudent(String studentId) returns, for the given student, the name of the exams for which he got a pass grade, grouped by grade; the grades are sorted in descending order, the exams appear in alphabetic order.

The method gradesOfCourse(String courseName), for the given course, groups by grade the ids of the students that passed the relative exam; the grades are in decreasing order, the student ids in alphabetic order.

The method failingGradesOfStudent(String studentId) returns the number of the failed exams for the given student.

The method failingGradesOfCourse(String courseName) returns the number of the fail grades that has been assigned for the given course.

The above four methods throw an exception if the student id or the course name are undefined.
